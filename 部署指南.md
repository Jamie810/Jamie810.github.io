# 个人主页部署指南

本指南将帮助你将个人主页部署到互联网上，让全世界的人都能访问。

## 🚀 推荐部署方式（免费）

### 方式一：GitHub Pages（推荐，最简单）

GitHub Pages 是 GitHub 提供的免费静态网站托管服务，非常适合个人主页。

#### 步骤：

1. **创建 GitHub 账号**
   - 访问 https://github.com
   - 如果没有账号，先注册一个

2. **创建新仓库**
   - 登录 GitHub
   - 点击右上角 "+" → "New repository"
   - 仓库名建议：`你的用户名.github.io`（例如：`zhangsan.github.io`）
   - 设置为 Public（公开）
   - 勾选 "Add a README file"
   - 点击 "Create repository"

3. **上传文件到 GitHub**
   
   **方法A：使用网页上传（最简单）**
   - 在仓库页面，点击 "uploading an existing file"
   - 将你的 `index.html`、`styles.css`、`script.js` 文件拖拽上传
   - 点击 "Commit changes"

   **方法B：使用 Git 命令行**
   ```bash
   # 在项目文件夹中打开终端
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/你的用户名/你的仓库名.git
   git push -u origin main
   ```

4. **启用 GitHub Pages**
   - 在仓库页面，点击 "Settings"（设置）
   - 左侧菜单找到 "Pages"
   - 在 "Source" 下选择 "main" 分支
   - 点击 "Save"
   - 等待几分钟，GitHub 会给你一个网址：`https://你的用户名.github.io`

5. **访问你的网站**
   - 访问 `https://你的用户名.github.io`
   - 如果看不到，等待几分钟后刷新

#### 优点：
- ✅ 完全免费
- ✅ 使用 GitHub 子域名
- ✅ 自动 HTTPS
- ✅ 可以绑定自定义域名

---

### 方式二：Netlify（推荐，功能强大）

Netlify 提供免费的静态网站托管，支持拖拽部署，非常方便。

#### 步骤：

1. **访问 Netlify**
   - 访问 https://www.netlify.com
   - 使用 GitHub 账号登录（或注册新账号）

2. **部署网站**
   
   **方法A：拖拽部署（最简单）**
   - 登录后，将整个项目文件夹拖拽到 Netlify 的部署区域
   - 等待上传完成
   - Netlify 会自动生成一个网址，例如：`https://random-name-123.netlify.app`

   **方法B：通过 GitHub 连接**
   - 点击 "Add new site" → "Import an existing project"
   - 选择 "Deploy with GitHub"
   - 授权并选择你的仓库
   - Netlify 会自动部署

3. **自定义域名（可选）**
   - 在网站设置中，点击 "Domain settings"
   - 可以修改默认的 Netlify 域名
   - 也可以绑定你自己的域名

#### 优点：
- ✅ 完全免费
- ✅ 拖拽即可部署，无需 Git
- ✅ 自动 HTTPS
- ✅ 支持自定义域名
- ✅ 自动部署（连接 GitHub 后）

---

### 方式三：Vercel（推荐，速度快）

Vercel 是另一个优秀的静态网站托管平台，部署速度快。

#### 步骤：

1. **访问 Vercel**
   - 访问 https://vercel.com
   - 使用 GitHub 账号登录

2. **部署网站**
   - 点击 "Add New Project"
   - 选择你的 GitHub 仓库（或直接上传文件夹）
   - 点击 "Deploy"
   - 等待部署完成，会得到一个网址：`https://你的项目名.vercel.app`

#### 优点：
- ✅ 完全免费
- ✅ 部署速度快
- ✅ 自动 HTTPS
- ✅ 支持自定义域名

---

### 方式四：Cloudflare Pages（推荐，全球加速）

Cloudflare Pages 提供全球 CDN 加速，访问速度快。

#### 步骤：

1. **访问 Cloudflare Pages**
   - 访问 https://pages.cloudflare.com
   - 使用邮箱注册账号

2. **部署网站**
   - 点击 "Create a project"
   - 连接 GitHub 仓库或直接上传文件
   - 点击 "Deploy site"
   - 会得到一个网址：`https://你的项目名.pages.dev`

#### 优点：
- ✅ 完全免费
- ✅ 全球 CDN 加速
- ✅ 自动 HTTPS
- ✅ 支持自定义域名

## 📝 部署前检查清单

在部署之前，确保：

- [ ] 所有个人信息已更新（名字、邮箱等）
- [ ] 所有链接都已检查（论文链接、项目链接等）
- [ ] 图片和资源文件路径正确
- [ ] 联系方式信息准确
- [ ] 在浏览器中测试过，显示正常

## 🔧 常见问题

### Q: 部署后看不到更新？
A: 清除浏览器缓存，或等待几分钟后刷新。GitHub Pages 可能需要几分钟才能更新。

### Q: 如何绑定自己的域名？
A: 
1. 在域名注册商处购买域名
2. 在部署平台（GitHub Pages/Netlify等）的设置中添加自定义域名
3. 按照平台提示配置 DNS 记录

### Q: 如何更新网站内容？
A: 
- **GitHub Pages**: 修改文件后，提交到 GitHub，会自动更新
- **Netlify/Vercel**: 如果连接了 GitHub，提交代码后自动更新；如果手动上传，需要重新上传文件

### Q: 网站加载慢怎么办？
A: 
- 使用 Cloudflare Pages（全球 CDN）
- 优化图片大小
- 压缩 CSS 和 JavaScript 文件
